Themen für die Klassenarbeit

Abkürzungen in der Netzwerktechnik

  • IEEE - Institute of Electrical and Electronics Engineers: Ein Berufsverband für Ingenieure, der Standards in der Elektrotechnik entwickelt.
    Beispiel: Der IEEE 802.11-Standard definiert WLAN-Technologien.
  • IETF - Internet Engineering Task Force: Eine Organisation, die Internetstandards wie TCP/IP entwickelt.
    Beispiel: Die IETF arbeitet an Protokollen für das Internet der Dinge.
  • RFC - Request for Comments: Dokumente, die neue Forschungsergebnisse, Innovationen oder Methoden in Bezug auf das Internet beschreiben.
    Beispiel: RFC 791 definiert das Internet Protocol Version 4 (IPv4).
  • ISO - International Organization for Standardization: Eine internationale Normungsorganisation, die weltweit technische Standards entwickelt.
    Beispiel: Das ISO/OSI-Modell ist ein Referenzmodell für Netzwerkprotokolle.
  • OSI - Open Systems Interconnection: Ein Schichtenmodell zur Beschreibung von Netzwerkkommunikation.
    Beispiel: Das OSI-Modell besteht aus sieben Schichten von der physikalischen bis zur Anwendungsschicht.

Kopplungsgeräte

  • Repeater: Ein Gerät, das Signale verstärkt, um größere Distanzen zu überbrücken.
    Beispiel: Ein Repeater wird verwendet, um ein schwaches WLAN-Signal zu verstärken.
  • Hub: Ein einfaches Netzwerkgerät, das Daten an alle angeschlossenen Geräte sendet.
    Beispiel: Ein Hub verbindet mehrere Computer in einem LAN, ohne den Datenverkehr zu filtern.
  • Bridge: Verbindet zwei Netzwerksegmente auf Datenverbindungsebene und filtert den Verkehr.
    Beispiel: Eine Bridge kann zwei Ethernet-Netzwerke verbinden und Daten basierend auf MAC-Adressen weiterleiten.
  • Switch: Ein Gerät, das den Datenverkehr effizient an das richtige Ziel im Netzwerk leitet.
    Beispiel: Ein Switch leitet Daten nur an den Port weiter, an dem das Zielgerät angeschlossen ist.
  • Router: Leitet Datenpakete zwischen verschiedenen Netzwerken auf Basis von IP-Adressen weiter.
    Beispiel: Ein Router verbindet ein Heimnetzwerk mit dem Internet.
  • Layer 3 Switch: Kombiniert die Funktionen eines Switches und Routers, arbeitet sowohl auf der Datenverbindungs- als auch auf der Netzwerkschicht.
    Beispiel: Ein L3-Switch kann VLANs routen und somit interne Netzwerksegmentierung ermöglichen.

ISO/OSI-Schichtenmodell

Das ISO/OSI-Modell ist ein Referenzmodell mit sieben Schichten, das die Kommunikation in Netzwerken beschreibt:

  1. Bitübertragungsschicht (Physikalische Schicht): Überträgt Bits über physische Medien.
    Beispiel: Ethernet-Kabel übertragen elektrische Signale.
  2. Sicherungsschicht (Datenverbindungsschicht): Sichert die Datenübertragung und steuert den Zugriff auf das Medium.
    Beispiel: MAC-Adressen werden auf dieser Schicht verwendet.

Strukturierte Verkabelung

Eine standardisierte Methode zur Verkabelung von Gebäuden und Rechenzentren, die Flexibilität und Skalierbarkeit ermöglicht. Sie wird in Unternehmensnetzwerken eingesetzt, um effiziente und organisierte Verbindungen zu gewährleisten.

Beispiel: In einem Bürogebäude werden alle Netzwerkdosen nach einem einheitlichen Plan verkabelt.

Netztopologien

Die physische oder logische Anordnung von Netzwerken:

  • Stern-Topologie: Alle Geräte sind an einen zentralen Knoten angeschlossen.
    Beispiel: Ein Switch in der Mitte verbindet mehrere Computer.
  • Bus-Topologie: Alle Geräte teilen sich ein gemeinsames Übertragungsmedium.
    Beispiel: Ältere Ethernet-Netzwerke mit Koaxialkabel.

Ethernet-Frame

Ein Datenpaket auf der Sicherungsschicht mit folgenden Feldern:

  • Präambel: Synchronisation für den Empfänger.
  • Ziel-MAC-Adresse
  • Quell-MAC-Adresse
  • Typ/Length
  • Daten
  • Frame Check Sequence (FCS): Fehlerprüfung.

Spanning Tree Protocol (STP)

Ein Protokoll zur Vermeidung von Schleifen in redundanten Netzwerkstrukturen.

  • Root Port: Port mit dem kürzesten Pfad zur Root Bridge.
  • Designated Port: Port, der als Ausgangspunkt zu einem Netzwerksegment dient.
  • Blocked Port: Port, der deaktiviert ist, um Schleifen zu verhindern.

Bridge Protocol Data Unit (BPDU): Nachrichten, die zum Austausch von Informationen zwischen Switches verwendet werden.

Übertragungsarten

  • Broadcast: Nachricht an alle Geräte im Netzwerk.
    Beispiel: ARP-Anfragen werden als Broadcast gesendet.
  • Multicast: Nachricht an eine Gruppe von Geräten.
    Beispiel: Video-Streaming an mehrere Empfänger.
  • Unicast: Nachricht an ein einzelnes Gerät.
    Beispiel: Eine E-Mail an einen einzelnen Empfänger.
  • Anycast: Nachricht an das nächstgelegene Gerät aus einer Gruppe.
    Beispiel: DNS-Anfragen nutzen Anycast-Adressen.
